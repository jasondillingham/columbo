# v0.4 plan — parallel lane orchestration + reconciliation (dedup)

Milestone (DESIGN.md): *Reconciliation + parallel lane orchestration locally.*

Read `docs/v0.3-plan.md` first. v0.3 shipped three lanes + the writer, run
sequentially by `columbo audit round`, emitting raw per-probe findings (the
Leonard round produced 28 findings that collapse to ~3 real classes). v0.4
makes the round run lanes concurrently and collapse the duplication.

## What "reconciliation" means here (the scope call)

The seed `reconcile.py` solves ID *collisions*: parallel red-team agents were
separate processes each self-numbering `F009..`, so they clashed and the
reconciler renumbered the losers into free ranges. **Columbo does not have
that problem locally** — `findings.Round.AssignIDs` numbers centrally after
all lanes return, so goroutine lanes never collide. The ID-collision merger is
only needed once lanes self-number as separate processes (the k3s-job model,
v0.6). It is explicitly NOT v0.4.

The reconciliation that *is* valuable locally is **finding deduplication**:
the 28-finding Leonard round is really "11 tools leak on null strings, 9 leak
on INT64-max, 1 uses code:0." DESIGN files *embedding-based* dedup under v0.7
and calls the hand-written version "the regex-and-prayer reconciler we wrote
by hand for Leonard and bosun" — i.e. a structural dedup is the documented
baseline that v0.7 upgrades. So v0.4 builds the structural baseline; v0.7
swaps in embeddings later.

## Two deliverables

### 1. `internal/orchestrator/` — parallel lane fan-out

Run the round's lanes concurrently instead of sequentially. Each lane is a
thunk `func() []lanes.Result`; the orchestrator runs them in goroutines and
returns results in a **deterministic lane order** regardless of finish order
(index the goroutines, collect into a fixed-size slice). Bounded trivially by
the lane count (3 today, ≤8 ever).

- Wall-clock win: L1 is dominated by the target's `go test ./...` (~minutes);
  L2 fires ~95 sequential probes (~30s); L6 is quick. Sequential = sum;
  parallel = max(L1). The L2/L6 time hides under L1.
- Concurrency safety: lanes each build the MCP server to their OWN temp dir
  (`go build -o <distinct>`), and `go build`/`go test`/`go run` share Go's
  build+module caches, which are concurrency-safe. Lanes spawn isolated
  server processes. No shared mutable state between lanes.
- Parallelism is ACROSS lanes only; each lane stays internally sequential
  (RunL2 loops its probes). 3 concurrent lanes, each sequential inside. Modest
  and safe — the abstraction is what pays off at v0.6 (8 k3s pods).

### 2. `internal/reconcile/` — structural finding dedup

Collapse findings of the same class within a lane into one finding that lists
the affected sites. Deterministic, not "prayer": probes emit an explicit
**class key**, and reconcile groups by it. No regex-guessing at titles.

- Add `Class string` to `findings.Finding`. Probes set it:
  - caps null-string-leak -> `"reflect-null-leak"` (F003 class)
  - caps INT64-overflow-leak -> `"int64-overflow-leak"` (F002 class)
  - protocol code:0 -> `"jsonrpc-code-zero"` (F004 class)
  - L1 version drift, etc. -> their own stable classes
- `reconcile.Dedup(fs []Finding) []Finding`: group by `(Severity, Class)`.
  A group with >1 member and a non-empty Class collapses into one Finding:
  - Title: the representative title, normalized + ` (N instances)`.
  - Files: union of all members' Files.
  - Observed: a compact list of the collapsed loci (tool/field), capped with a
    "+K more" if long, never silently truncated.
  - Reproducer: keep ONE representative member's reproducer verbatim (audit
    format requires a runnable reproducer per finding; the Observed loci list
    makes the other instances recoverable). Do not concatenate or drop it.
  - Severity/Class: the group's.
  - Findings with an empty Class, or singleton groups, pass through unchanged.
- Applied **per-lane** in `audit round` (`report.Findings = Dedup(report.Findings)`)
  BEFORE `AssignIDs`, so the deduped set gets the F001.. numbering and per-lane
  files + consolidated rollup stay coherent (both show the deduped findings).
- Within-lane only for v0.4. The 27 caps findings are all in L2, so within-lane
  dedup solves the motivating case completely. Cross-lane dedup (same class
  surfaced by two lanes) is rarer; deferred with a note.

## Wiring

- `columbo audit round` runs lanes via the orchestrator (parallel), then
  `reconcile.Dedup` per lane, then assembles + writes the Round as today.
- Single-lane commands (`audit l1/l2/l6`) stay raw (no dedup) — a focused lane
  run wants every instance. Dedup is a campaign-level (round) step.
- Add a `--raw` flag to `audit round` to skip dedup (escape hatch for seeing
  every instance). Default is reconciled.

## Build sequence

0. **Baseline first.** Before swapping in the orchestrator, capture the
   current sequential `audit round --raw` findings on Leonard (IDs, titles,
   files, order) as a reference file. This is the diff target for step 4's
   gate.
1. `internal/orchestrator/orchestrator.go` — `RunParallel(thunks) [][]Result`
   (ordered). Unit-test with synthetic thunks (out-of-order completion still
   returns in order; all run concurrently).
2. `internal/reconcile/reconcile.go` — `Dedup`. Unit-test: a class with N
   members collapses to 1 with a unioned Files list and an instance count;
   empty-Class and singletons pass through; severity preserved.
3. `findings.Finding.Class` + probes set it (caps, protocol, L1). Adjust the
   `classify*` functions; extend their tests to assert Class is set.
4. Wire `audit round` to orchestrate-then-dedup; `--raw` opt-out.
5. Verify end-to-end against Leonard. **The gate is a DIFF, not a count:**
   parallel `--raw` must equal the step-0 sequential baseline — same IDs,
   titles, files, same order — not merely "28 findings." This is the one thing
   goroutine fan-out can actually break. Specific risk: three concurrent
   `leonard-mcp` processes against the same `.leonard/` SQLite store. The
   write-probes (`record_claim`/`record_decision`/`supersede_decision` with
   malformed args) should fail schema-validation before any DB write, keeping
   concurrent access read-only — but verify it. A clean diff proves it; a
   "database is locked" error or a reordered ID means concurrency changed the
   result and must be fixed (e.g. serialize, or per-lane store clones).

## v0.4 "done"

- `columbo audit round examples/leonard.target.yaml` runs L1+L2+L6 in
  parallel and writes a round whose L2 findings are deduped (27 -> ~2 with
  affected-tool lists), coherent across per-lane and consolidated docs.
- `--raw` reproduces the v0.3 un-deduped output (28).
- Orchestrator and reconcile are unit-tested; the `classify*` Class additions
  are tested; `make check` stays green.

## Note: black-box findings satisfy the format via reproducer, not Files

L2/L6 findings observe wire behavior and cannot attribute a target source file,
so they carry no `Files`. `audits/README.md`'s promotion criteria name
"source-file references via repo-relative paths" — for black-box probes the
runnable reproducer IS the evidence, and that's what's kept (verbatim, even
through dedup). This is defensible, not a gap; recorded so a future reader
doesn't mistake the empty Files for an omission. A source-attribution pass
(grep the leaked struct/symbol back to its definition) is possible future work,
not v0.4.

## Explicitly deferred

- **ID-collision merger** (the seed's actual reconcile.py job): only needed
  when lanes self-number as separate processes — k3s jobs, v0.6.
- **Embedding-based dedup**: v0.7 (local model on Thor). v0.4 is the
  structural baseline it replaces.
- **Cross-lane dedup**: within-lane covers the motivating case; revisit if a
  class genuinely spans lanes.
- **k3s job orchestration**: v0.6. v0.4 orchestration is local goroutines.
