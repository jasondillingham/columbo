# v1.0 plan — self-audit: bughunt-1 against columbo, clean

Milestone (DESIGN.md): *Self-audit (bughunt-1 against columbo) clean.* The
proof-of-life: "If Columbo can't find bugs in its own probe library, it isn't
earning its keep." Read `docs/v0.7-plan.md` and `audits/README.md` first.

## The honesty trap to avoid

Running L1/L2/L6 against `columbo-mcp` and declaring "clean" is HOLLOW — that
surface was already the v0.5 discrimination test (6 PASS / clean), and the
automated lanes only cover the MCP surface + build invariants, not the
codebase. A self-audit that only re-runs the lanes proves nothing new. So
bughunt-1 has TWO sources, matching how Leonard's bughunt-12 and bosun's
bughunt-1 actually ran (Driven mode, Claude in the loop):

1. **Automated lanes** (L1 build, L2 caps, L6 protocol) against
   `examples/columbo.target.yaml`. The machine half. Expected mostly clean.
2. **Driven adversarial code review** — the cross-file reasoning DESIGN says is
   a frontier-model job, NOT the local model or the lanes. This is where the
   real findings come from: read columbo's own code adversarially, especially
   the paths that handle untrusted/adversarial input and concurrency (ironic
   for an auditor to get wrong), and the issues already surfaced this session.

## What "clean" means (so we don't fake it)

Not "no bugs exist" — that's unprovable. Clean = the round's findings are
triaged and the serious ones closed, exactly how Leonard/bosun bughunts close:

- Every **CRITICAL/HIGH**: fixed, with the fix in the same round.
- **MEDIUM/LOW**: fixed when cheap, otherwise recorded with explicit rationale
  (status `confirmed` + a deferral note), never silently dropped.
- The round lands as `audits/bughunt-1-*.md` (dogfood columbo's own writer for
  the rollup; hand-author the detail sections the writer leaves as TODO).

## Review hotspots (where real bugs hide)

Prioritize the adversarial-input + concurrency + correctness surfaces:

- `internal/probes/mcp/client.go` — the transport. The **F018-class unbounded
  `ReadBytes`** is already a KNOWN self-finding (documented in v0.5-deferred):
  columbo's own MCP CLIENT can be OOM'd by a no-newline flood. File it.
- `internal/mcpserver/server.go` — bounded reader, error codes (built to spec,
  but verify the bound actually holds and notifications/edge frames).
- `internal/findings/writer.go` — table-cell escaping, the parse-back
  round-trip, ID assignment edge cases.
- `internal/reconcile/` — structural over-merge on shared Class (the LLM-probe
  interaction) and embed single-linkage chaining (both already characterized).
- `internal/k3srunner/` + `internal/autonomous/promote.go` — Job lifecycle,
  git ops, the dirty-tree/clean-tree invariants.
- `cmd/columbo/main.go` — the `auto --k3s` round-number-0 bug (already known),
  flag wiring, stale-binary footguns.
- `internal/lanes/*` — classifier correctness, the build-artifact-dirties-tree
  interaction.

Known issues already surfaced this session (must appear in the round, fixed or
deferred-with-rationale): F018 in the MCP client; `auto --k3s` hardcodes Job
round number 0; embed single-linkage can chain; ldflags-vs-`go run` P3 gap
(v0.2-deferred); the build-artifact dirty-tree interaction.

## Sequence

1. Run the automated lanes against `examples/columbo.target.yaml`; capture.
2. Driven review of the hotspots; collect findings (severity, files,
   reproducer, fix shape) — be adversarial, prefer concrete reproducers.
3. Write `audits/bughunt-1-*.md` (brief + per-lane/per-area + consolidated
   rollup). Use the writer where it fits.
4. Fix CRITICAL/HIGH (and cheap MEDIUM/LOW); for the rest, status + rationale.
5. `make check` stays green after fixes; re-run affected lanes to confirm.
6. Commit the round + fixes.

## v1.0 "done"

- `audits/bughunt-1-*.md` exists: a real round with both lane findings and
  reasoning findings, in audit-format, IDs assigned, severities triaged.
- No open CRITICAL/HIGH (closed in-round); MEDIUM/LOW closed or
  deferred-with-rationale.
- `make check` green; the automated lanes against columbo-mcp clean.
- The round is honest: it does NOT claim columbo is bug-free, only that
  bughunt-1's findings are triaged and the serious ones fixed.

## Deferred (explicitly NOT claimed by v1.0)

- A guarantee of bug-freeness. v1.0 closes bughunt-1's findings, not all bugs.
- Lanes beyond L1/L2/L6; deep audit of every package.
- The reasoning pass as an automated capability (it's Driven / human-in-loop).
