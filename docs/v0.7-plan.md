# v0.7 plan — local-model integration on Thor (probe-gen + embedding dedup)

Milestone (DESIGN.md): *Local-model integration on Thor for probe generation +
finding dedup.* Read `docs/v0.6-plan.md` first.

Infra is already up (probed 2026-06-01): Ollama on **thor** (10.66.0.45, the
RX 6700 XT box) with `qwen2.5-coder:7b/14b`, `llama3.1:8b`, `gemma4:26b`, and
embedding models `mxbai-embed-large` / `nomic-embed-text`. Reachable on the LAN.
No setup to build, unlike v0.6.

## The discipline shift this milestone forces

v0.2-v0.6 leaned on a hard diff-gate: parallel == sequential, cluster ==
local, byte-identical. **A local model makes output nondeterministic**, so that
gate no longer applies to LLM paths. Verification shifts to STRUCTURAL
properties:
- probe-gen: "produced valid, schema-shaped JSON args that actually ran"
  (not "produced these exact args").
- embedding dedup: "clustered the known 27 leonard leaks to the same ~2 groups
  structural dedup produces" — i.e. validated against the deterministic
  baseline as ground truth, asserting it does NOT over-merge.

Crucially: **the LLM is opt-in, defaults stay deterministic.** Fixed probes and
structural dedup remain the default, so `make check` and the existing
round/auto behavior are unchanged unless a flag asks for the model. The model
augments; it does not replace.

## Pieces to build

### 1. `internal/ollama/` — HTTP client

A small client for Ollama's HTTP API (the Go binary calls it at runtime; the
agent's MCP tools don't help the shipped tool):
- `Generate(model, prompt) (string, error)` -> `/api/generate` (stream:false).
- `Embed(model, text) ([]float32, error)` -> `/api/embed`.
- Host from `--ollama` flag / `COLUMBO_OLLAMA` env (default the thor URL);
  empty host = feature disabled (clean fallback).
- **Real timeout + fail-OPEN.** gemma4:26b on a 6700XT is slow; a model that's
  down or stalls must drop back to fixed probes / structural dedup, NEVER block
  or fail the audit. The model is an augmentation; its absence is not an error.
- Unit-tested against an `httptest` fake server (deterministic: assert request
  shape + response parsing). **No `make check` test may reach thor** — live
  Generate/Embed checks are throwaway/guarded (like the old `TestRealLeonard`);
  only the httptest fakes and synthetic-vector cluster tests run in the suite.

### 2. Embedding dedup (the verifiable half)

DESIGN: "sentence-transformer embeddings beat the regex-and-prayer reconciler."
Add `reconcile.DedupEmbed(client, model, findings, threshold)`:
- Embed each finding's Title (+ Class) via `mxbai-embed-large`.
- Agglomerate by cosine similarity above a threshold; collapse each cluster the
  same way structural merge does (representative + locus list + unioned files +
  one reproducer).
- **Threshold: loss is ASYMMETRIC, so do NOT tune-to-fit leonard.** For an
  auditor, under-merging costs a duplicate row a human skims; over-merging
  HIDES a real bug. Not symmetric. Pick a conservative (high-similarity)
  threshold that merges only near-identical titles, accept it under-merges vs
  structural's exact-key sometimes, and let the asymmetry — not leonard's 2
  clusters — set it.
- Wired behind `--dedup=embed` (default `structural`).

**The differentiator test is the gate (leonard can't show it — it ties).** On
leonard, structural and embed both yield ~2 clusters, so leonard proves only
"doesn't break," not value. The case that proves value AND guards a real bug:
several DISTINCT findings that happen to share a `(Severity, Class)`. Structural
dedup merges any same-key group >1, so it OVER-MERGES them into one finding (the
others become a false "+N more sites" — the hidden-finding sin). Embedding dedup
clusters by title semantics, so it keeps distinct titles separate. One synthetic
test asserts exactly that: structural over-merges the distinct-but-same-class
set; embed keeps them apart. This proves embed's differentiator and is the GATE
before piece 3 (which would otherwise ship the over-merge bug — see below).

### 3. LLM probe generation (the exploratory half)

DESIGN: "give an LLM the target's input schema, ask for adversarial JSON."
Add `caps.GenerateLLM(client, model, tool, n)`:
- Prompt a coder model (`qwen2.5-coder:7b`) with the tool's inputSchema, ask
  for N adversarial argument objects (oversized, type-confused, boundary,
  injection-shaped), parse the JSON, turn each into a Probe.
- **Class must be empty (or per-finding unique), NOT a shared "llm-generated".**
  Structural dedup merges any same-`(Severity,Class)` group >1, so a shared
  class would over-merge distinct LLM findings into one (hidden-finding sin).
  Empty Class passes through structural unmerged; embedding dedup then merges
  only the genuinely-same ones by title. (This is the bug the piece-2 gate
  guards.)
- **Honesty firewall:** the LLM PROPOSES inputs; the deterministic, tested
  `classifyL2` DISPOSES — the model never decides what's a finding. The
  reproducer must capture the concrete generated `Args` (Probe.Args already
  holds them), never "re-prompt the model," or it isn't reproducible.
- Wired behind `--llm-probes[=N]` on L2 (default off). Fixed battery still runs.
- Verification (nondeterministic): the model returns parseable arg objects that
  run against leonard-mcp and classify cleanly. Surfacing a NEW finding is the
  aspiration, NOT the gate — and STATUS must not dress "the model emitted JSON"
  as a capability if it found nothing new on leonard.

## Build sequence

1. `internal/ollama/` client + httptest unit tests. Live smoke vs thor
   (`Generate` returns text, `Embed` returns a vector of expected dim).
2. Embedding dedup + tests: unit-test the cosine/cluster logic with synthetic
   vectors (deterministic — no model needed); then a live run on leonard's 27
   leaks asserting ~2 clusters, matching structural (the no-over-merge bar).
3. LLM probe-gen + tests: unit-test JSON parsing with a fake `Generate`; live
   run vs leonard-mcp producing valid runnable probes.
4. Wire `--ollama`, `--dedup=embed`, `--llm-probes` flags. Defaults unchanged.

## v0.7 "done"

- `internal/ollama` client works against thor (Generate + Embed), unit-tested
  against a fake.
- `--dedup=embed` on a leonard round reproduces structural's ~2 clusters (no
  over-merge), validated against the deterministic baseline.
- `--llm-probes` produces valid schema-aware adversarial probes that run
  against leonard-mcp.
- Defaults stay deterministic; `make check` green and unchanged behavior
  without the flags.
